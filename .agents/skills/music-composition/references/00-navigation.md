# Navigation Map

This file routes user requests to the specific reference files you need. **Read this before diving into any other reference**, every time you use this skill.

## How to use this map

1. Find the section matching the user's request (quick-lookup table, vague-problem table, or genre table).
2. Load the listed file(s).
3. If the request crosses domains, load primary files for each — but resist loading more than 3.
4. If you would need to load 5+ files, the question is too broad. Pick the dominant aspect and answer that first; offer to dig into adjacent topics afterward.

## Vague creative problem → diagnostic

When the user describes a feeling, problem, or fuzzy goal, translate it into a technical diagnosis before answering. Common patterns:

| User says | Likely diagnostic | Files to consult |
|-----------|-------------------|------------------|
| "It feels boring" | Predictable harmony / rhythm / dynamics | `harmony/chromatic-harmony.md`, `rhythm-groove/rhythmic-devices.md`, `production-aware/energy-and-dynamics.md` |
| "The chorus doesn't hit" | Insufficient lift from verse — register, density, harmony, rhythm | `form/narrative-and-transitions.md`, `orchestration/arrangement-density.md`, `songwriting/hooks-and-memorability.md` |
| "It feels generic / too 'pop'" | Default progressions, predictable form, overused voicings | `harmony/reharmonization.md` + relevant genre file |
| "The transition is weak / awkward" | Insufficient or excessive setup; abrupt key/density change | `form/narrative-and-transitions.md`, `harmony/modulation.md` |
| "The melody is forgettable" | No motivic identity, weak contour, no memorable interval | `melody/melodic-construction.md`, `melody/motivic-development.md`, `songwriting/hooks-and-memorability.md` |
| "It sounds muddy / cluttered" | Voicing, register, density, frequency overlap | `orchestration/voicing-and-texture.md`, `orchestration/arrangement-density.md`, `production-aware/arrangement-for-mix.md` |
| "It needs more emotion / depth" | Often: harmony too plain, dynamics too flat, melody too narrow | `harmony/chromatic-harmony.md`, `production-aware/energy-and-dynamics.md`, `melody/melodic-construction.md` |
| "It's stuck on one idea" | No development, no variation | `melody/motivic-development.md`, `techniques/theme-and-variation.md` |
| "It feels too predictable" | Harmonic and formal default-mode | `harmony/reharmonization.md`, `harmony/chromatic-harmony.md`, `form/narrative-and-transitions.md` |

## Multi-file scenarios

Some requests genuinely span several files. Common patterns:


**"Brainstorm song/cue ideas"** — `creative-workflows/musical-brainstorming.md` + `../assets/musical-brainstorming-cards.md` + relevant genre file if known. Generate divergent options first; converge only after the user reacts.

**"Let's keep developing this over multiple turns"** — `creative-workflows/user-agent-collaboration.md` + `../assets/session-brief-and-decision-log.md` + `workflow.md`. Maintain a project card and decision log when the session becomes stateful.

**"Revise this but keep [specific element]"** — `creative-workflows/revision-and-feedback-loops.md` + `../assets/session-brief-and-decision-log.md` + the relevant craft file. Preserve the liked element and change the smallest musical variable that can solve the problem.

**"Search the web for references/trends"** — `../assets/web-search-cheatsheet.md` + `research/web-music-trend-research.md` + `research/reference-track-digging.md`. Use the cheatsheet for query construction; use research files for interpretation and composition translation.

**"Write a song in [genre]"** — The genre file + `form/popular-song-forms.md` + `songwriting/hooks-and-memorability.md` + relevant harmony file (`harmony/functional-harmony.md` for most, `harmony/jazz-harmony.md` for jazz, `harmony/modal-harmony.md` if modal).

**"Analyze this piece"** — `analysis.md` + relevant harmony file(s) + `form/classical-forms.md` or `form/popular-song-forms.md` depending on style.

**"Reharmonize this"** — `harmony/reharmonization.md` + `harmony/chromatic-harmony.md` + (if jazz/contemporary) `harmony/jazz-harmony.md`. Reharm techniques cross-pollinate.

**"Make this section more dramatic / climactic"** — `form/narrative-and-transitions.md` + `orchestration/arrangement-density.md` + `production-aware/energy-and-dynamics.md` + `harmony/modulation.md`.

**"Compose for [ensemble or solo instrument]"** — `orchestration/instruments-ranges-character.md` + the relevant `instrument-idiom/` file + the relevant genre file + `orchestration/voicing-and-texture.md`.

**"Make this part playable / idiomatic"** — relevant `instrument-idiom/` file + `orchestration/instruments-ranges-character.md` + `orchestration/voicing-and-texture.md` if part density matters.

**"Write a [solo / improv / lead line] over these changes"** — `harmony/jazz-harmony.md` (for any non-trivial changes) + `melody/melodic-construction.md` + the relevant genre file.

**"Make this groove harder / tighter / weirder"** — `rhythm-groove/groove-and-feel.md` + `rhythm-groove/rhythmic-devices.md` + the relevant genre file.

**"What is current / trending in [genre/market]?"** — `research/web-music-trend-research.md` + the relevant genre file + `research/reference-track-digging.md` if the answer should become compositional advice.

**"Make it sound like [artist/song]"** — `research/style-reference-and-copyright.md` + `research/reference-track-digging.md` + the relevant genre file. Extract variables; do not clone melody, lyrics, riffs, samples, vocal identity, or signature production tags.

## Loading discipline

This skill is large. Load minimally:

- **One reference file** is often enough for a focused question.
- **Two to three** for cross-domain or genre-specific questions.
- **Loading 5+ files** suggests the question is too broad — narrow it first.

When in doubt: pick the file most directly addressing the user's *immediate* need. Answer that. Then offer to dig into adjacent topics if they want.

If a file you need doesn't exist yet, say so honestly: "This skill doesn't cover [X] yet — I'll work from general principles and flag what I'm uncertain about." Then do that. Don't fabricate a confident answer.

## When to ask the user before loading

Ask before loading if:

- The genre is critical and unclear ("write a chord progression for a sad bridge" — pop? jazz? cinematic?).
- The user's level is critical and unclear (a beginner asking about "modes" needs different depth than a jazz player asking about "modal harmony").
- The request is ambitious and could go in opposing directions ("write a song" — you need to know length, genre, instrumentation, lyrical theme, etc. before being useful).
